# ArchiGraph — Unity Architecture & Dependency Visualization Tool

ArchiGraph — инструмент для Unity Editor, предназначенный для анализа архитектуры проекта и визуализации зависимостей между классами и интерфейсами.  
Плагин строит интерактивный граф зависимостей, группирует типы по пространствам имён и предоставляет гибкие настройки отображения.


## Возможности

- Автоматическое сканирование классов и интерфейсов в выбранной папке.
- Анализ зависимостей: поля, свойства, параметры методов, конструкторы.
- Построение интерактивного графа зависимостей.
- Группировка нод по namespace.
- Настройка цветов и расстояний между нодами.
- Переключатель отображения зависимостей.
- Перестройка графа одной кнопкой.
- Работа как с папкой, так и со всем проектом.

## Установка

### Вариант 1 — через .unitypackage

1. Скачайте файл ArchiGraph.unitypackage.
2. В Unity выполните:
Assets → Import Package → Custom Package
3. Импортируйте содержимое.
4. Откройте инструмент:
Window → ArchiGraph

### Вариант 2 — из репозитория

1. Склонируйте проект или скачайте ZIP.
2. Скопируйте: Assets/Editor/ArchiGraph/
3. Запустите через меню: Window → ArchiGraph

## Использование

### Открытие инструмента

Window → ArchiGraph

### Построение графа для всего проекта

Кнопка: `Show All`

Выполняет:

- анализ всех типов в Assets;
- построение всех нод;
- обязательное отображение зависимостей.

### Построение графа для выбранной папки

Кнопка: `Select Folder`

Построение графа для файлов внутри выбранной директории.

### Панель свойств

Кнопка: `Add Properties`

Содержит:

- <b>Цвета</b>
    * Output Class Color
    * Output Interface Color
    * Input Color  

- <b>Layout</b>
    * Offset — расстояние между нодами (при изменении выполняется центрирование графа)

- <b>Управление зависимостями</b>
    * Show Dependency — включить или выключить линии зависимостей

- <b>Применение настроек</b>
    * Кнопка: `Apply`

## Управление графом

- Зум — колесо мыши  
- Перемещение камеры — ПКМ или средняя кнопка  
- Выбор ноды — ЛКМ  
- Информация о ноде отображается в Properties Node


## Принцип работы

1. Сканируются C# классы и интерфейсы.
2. Определяются зависимости:
   - поля
   - свойства
   - параметры методов
   - конструкторы
3. Создаются ноды.
4. Между нодами создаются линии (если Show Dependency включён).
5. Ноды группируются по namespace.
6. При необходимости граф центрируется.

## Требования

- Unity 2021.3 или новее  
- Работает только в Editor (не влияет на сборки)
